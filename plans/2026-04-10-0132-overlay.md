# Overlay Implementation Plan

**Last Updated:** 2026-04-10
**Status:** Planned
**Supersedes:** `overlay-followup-fixes-plan.md`, `blob-overlay-plan.md`

## Summary

Four phases of overlay and tray polish, with the blob overlay as the final visual target:

1. **Tray exclusivity** — radio-group semantics for single-choice tray menus
2. **Multi-monitor reliability** — cursor-targeted placement stays on the right screen
3. **Transcript state** — show recognized text and clipboard ownership after transcription
4. **Blob overlay** — replace the corner card with a full-screen dimming overlay and animated centered blob

Phases 1–3 are backend/logic fixes. Phase 4 is the new visual design; it consumes the `TRANSCRIBED` state from Phase 3.

## Phase 1: Tray Option Exclusivity

### Goal

Single-choice tray submenus (language, position, display, auto-hide, density) render and behave as radio groups in both backends.

### Changes

**`whisper_tray/tray/menu.py` — `create_menu()` (pystray)**

Add `radio=True` to every `MenuItem` inside mutually exclusive submenus: language, position, display target, ready auto-hide, density. Keep overlay-enabled and auto-paste as plain checkboxes.

Note: `radio=True` changes the visual bullet marker only — functional exclusivity already works via per-item lambda callbacks. The flag is not supported by all pystray backends (macOS rumps); it is safe on Windows WinTray.

**`whisper_tray/tray/menu.py` — `create_qt_menu()` (Qt)**

Wrap each exclusive group in a `QActionGroup(menu, exclusive=True)` and call `group.addAction(action)` for each item. Replace the existing manual checked-state tracking for these groups.

### Tests (`tests/test_tray.py`)

- Add `FakeQtActionGroup` stub: `addAction(action)`, `setExclusive(bool)`, list of registered actions. Inject via existing `sys.modules` patching pattern.
- Assert pystray items in exclusive groups carry `radio=True`.
- Assert Qt grouped actions are registered in a `FakeQtActionGroup` with `setExclusive(True)`.
- Assert checked-state refresh still works after state changes (existing tests must still pass).

## Phase 2: Multi-Monitor Placement Reliability

### Goal

Cursor-targeted overlays reliably stay on the intended screen across display sleep/wake and transient geometry changes.

### Changes

**`whisper_tray/overlay/pyside_overlay.py` — `OverlayWindow` and `BlobOverlayWindow`**

`resolve_overlay_screen()` stays a pure helper (cursor or primary lookup — no changes). All richer fallback logic lives in `_reposition()`:

- Add `_last_resolved_screen: QScreen | None = None` to `__init__`.
- In `_reposition()`, update `_last_resolved_screen` whenever `resolve_overlay_screen()` returns a non-None result.
- When the lookup returns `None`, fall back in order: last resolved cursor screen → screen containing the current overlay geometry → primary screen.
- When `_last_resolved_screen` changes to a different screen object, reset `_last_anchor = None` to force a reposition (prevents stale coordinates from suppressing a needed move after display wake).

### Tests (`tests/test_overlay.py`)

- Cursor mode uses cursor screen when `screenAt(cursor)` returns a screen.
- Cursor mode reuses `_last_resolved_screen` when cursor lookup returns `None`.
- Fallback reaches primary only when both cursor lookup and `_last_resolved_screen` are `None`.
- `_last_anchor` is invalidated when `_last_resolved_screen` changes.

## Phase 3: Clipboard-Aware Transcript State

### Goal

After successful transcription, show the recognized text and whether it is still the clipboard content owned by WhisperTray. Keep it visible until a new recording starts, an error replaces it, or the clipboard is replaced by another source.

### New state: `AppState.TRANSCRIBED`

**`whisper_tray/state.py`**

- Add `TRANSCRIBED = "transcribed"` to `AppState`.
- Add `transcript: str | None = None` to `AppStateSnapshot` (raw recognized text; used for clipboard ownership comparison and passed to the presenter for display formatting).
- Add presenter logic for `TRANSCRIBED`:
  - badge: `"COPIED"` or `"PASTED"` based on whether auto-paste fired
  - primary: truncated transcript (compact: one line with ellipsis; detailed: up to 3 lines)
  - secondary: clipboard/paste status line (detailed only)
  - hint: `"Shown until clipboard changes"` (detailed only)
  - `icon_color`: `"lightgreen"`
  - `overlay_auto_hide_seconds`: `None` (TRANSCRIBED ignores `ready_auto_hide`; it stays until a trigger fires)
- Add `TRANSCRIBED` animation params to blob state table (see Phase 4).

**`whisper_tray/app.py`**

- In the transcription worker, change `next_state = AppState.READY` (success path) to publish `AppStateSnapshot(state=AppState.TRANSCRIBED, transcript=text)`.
- `_stop_flash_timer()` already calls `_publish_state(next_state)` — pass the full snapshot through instead of just the state enum.
- Transition back to `READY` when any of these occurs:
  - A new recording starts (hotkey pressed)
  - An error replaces TRANSCRIBED
  - `ClipboardManager.owns_clipboard()` returns `False` (checked on the next position-timer tick or auto-hide tick)

**`whisper_tray/clipboard.py`**

- Add `_last_owned_text: str | None = None`.
- Store `_last_owned_text = text` inside `copy_and_paste()` after `pyperclip.copy(text)`.
- Add `owns_clipboard() -> bool`: calls `pyperclip.paste()` and compares to `_last_owned_text` (returns `False` if `_last_owned_text is None`).
- Clipboard-change detection is lazy (poll on demand), not reactive. Qt `QClipboard.dataChanged` signal support is a follow-up.

### UX

- Detailed blob overlay:
  - badge: `COPIED` or `PASTED`
  - primary: transcript text (truncated with ellipsis)
  - hint: `"Shown until clipboard changes"`
- Compact blob overlay: primary line only, no hint.

### Tests

- `tests/test_state.py`: presenter tests for `TRANSCRIBED` (badge, primary truncation, hint, auto-hide=None, icon color).
- `tests/test_app.py`: successful transcription publishes `TRANSCRIBED` snapshot with correct `transcript` field.
- `tests/test_clipboard.py`: `owns_clipboard()` returns `True` after copy, `False` after external clipboard change, `False` before any copy.

## Phase 4: Blob Overlay

### Goal

Replace the corner-anchored card with a full-screen dimming overlay and a centered animated blob. The blob morphs and cycles color to communicate state visually; text labels appear below it.

### Visual design

```
┌─────────────────────────────────────────────────────┐
│░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░│  ← dim layer
│░░░░░░░                                    ░░░░░░░░░│
│░░░░░░░░░   ╭────────── blob ──────────╮   ░░░░░░░░░│
│░░░░░░░░   (   morphing colorful glow   )  ░░░░░░░░░│
│░░░░░░░░░   ╰───────────────────────────╯  ░░░░░░░░░│
│░░░░░░░                                    ░░░░░░░░░│
│                     LISTENING                       │  ← badge
│               Speak now, recording…                 │  ← primary
│               Ctrl+Shift+Space to stop              │  ← hint
│░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░│
└─────────────────────────────────────────────────────┘
```

### Window: `BlobOverlayWindow(QWidget)`

- Flags: `Tool | FramelessWindowHint | WindowStaysOnTopHint`
- Attributes: `WA_TranslucentBackground`, `WA_TransparentForMouseEvents`, `WA_ShowWithoutActivating`
- Geometry: full screen (`screen.geometry()`) via `resolve_overlay_screen()` (with Phase 2 fixes)
- `paintEvent`: fills entire surface with `rgba(0, 0, 0, dim_alpha)` — dim level varies per state

Layout (centered):
```
BlobOverlayWindow
└── QVBoxLayout (Qt.AlignCenter, no margins)
    ├── BlobWidget  (300×300 logical px)
    ├── QLabel#badge    (bold 13px, white)
    ├── QLabel#primary  (15px, white 90%)
    └── QLabel#hint     (12px, white 55%, hidden if None)
```

`overlay_secondary` is not shown — the blob communicates state visually.

### Widget: `BlobWidget(QWidget)`

Pure `QPainter` rendering. `QGraphicsBlurEffect(blurRadius=20)` applied to the widget.

**Shape algorithm (paintEvent):**

```
N = 10 control points at angles θ = 2π·i/N  (i = 0..N-1)
For each point:
    d = A₁·sin(2θ + t·ω₁) + A₂·sin(3θ + t·ω₂) + A₃·cos(5θ + t·ω₃)
    r = base_radius + d
    point = (cx + r·cos(θ),  cy + r·sin(θ))
Closed smooth spline through all points using QPainterPath.cubicTo()
```

Baseline amplitudes (scale with `base_radius`): `A₁=0.20r`, `A₂=0.10r`, `A₃=0.12r`  
Baseline frequencies: `ω₁=1.5`, `ω₂=2.3`, `ω₃=0.8`

**Color:**

Fill with `QRadialGradient`:
- Inner stop (r=0): `QColor.fromHsvF(hue, sat, 1.0, 0.9)` — hue cycles as `(t·25) % 360`
- Outer stop (r=1): `QColor.fromHsvF((hue+40)%360, sat·0.8, 0.7, 0.0)`

**State → parameters:**

| AppState | base_radius | amplitude scale | speed scale | saturation | dim alpha | hue |
|----------|-------------|-----------------|-------------|------------|-----------|-----|
| LOADING_MODEL | 60px | 0.06 | 0.4× | 0.3 | 100 | cycles |
| READY | 55px | 0.04 | 0.3× | 0.2 | 80 | cycles |
| RECORDING | 110px | 0.22 | 1.0× | 0.9 | 140 | cycles (fast) |
| PROCESSING | 80px | 0.10 | 0.6× | 0.7 | 120 | fixed ~30° (orange) |
| TRANSCRIBED | 65px | 0.05 | 0.35× | 0.6 | 90 | fixed ~120° (green) |
| ERROR | 70px | 0.08 | 0.5× | 0.8 | 130 | fixed 0° (red) |

DPI: multiply `base_radius` and widget size by `screen.devicePixelRatio()`.

**Animation timer:**

`_anim_timer: QTimer` at 33ms (~30fps). Each tick:
1. `_t += 0.033 * speed_scale`
2. Lerp `_current_radius` → `_target_radius` by `min(1.0, elapsed/0.3)` (300ms transition)
3. Lerp `_current_amplitude` → `_target_amplitude` same way
4. `blob_widget.update()`

Timer runs only while overlay is visible.

### Integration

**`whisper_tray/config.py`**

Add to `OverlayConfig`:
```python
overlay_style: str = "blob"   # "blob" | "card"
```
Read from `OVERLAY_STYLE` env var.

**`whisper_tray/overlay/pyside_overlay.py`**

`PySide6OverlayRuntime._create_window()` instantiates `BlobOverlayWindow` when `config.overlay_style == "blob"`, else `OverlayWindow`. Both implement the same `show_presentation(presentation)` / `close()` interface.

The existing `OverlayWindow` (card) is kept unchanged as the `card` style fallback.

| File | Change |

The original card surface rendering fixes (accent rail seam, border artifacts) are **not implemented** — the card style is retained only as a named fallback (`OVERLAY_STYLE=card`) for users who prefer it. No visual polish work is done on the card.

## Risks and Notes

- **QGraphicsBlurEffect at 30fps**: Blur recomputes every frame at 300×300px. Mitigation: drop to 30fps (already planned); or cache a pre-blurred base and animate only the color/shape layer on top.
- **Click-through on full-screen Windows**: Reuse existing `apply_windows_overlay_styles()` Win32 path (`WS_EX_TRANSPARENT`) unchanged. `WA_TransparentForMouseEvents` alone is unreliable on some compositors.
- **Blur + per-pixel alpha interaction**: Set `WA_TranslucentBackground` only on `BlobOverlayWindow` (top-level). `BlobWidget` should have an explicit transparent background but must not itself be a top-level window, to keep Qt's blur compositor working.
- **Clipboard ownership detection in headless tests**: `pyperclip.paste()` may not work on headless CI. Keep `owns_clipboard()` mockable via dependency injection or monkeypatch; do not call it from test code directly.
- **`_stop_flash_timer()` signature change**: Passing a full `AppStateSnapshot` instead of an `AppState` enum requires updating the call site in `app.py` and the type annotation. Verify no other callers assume the old signature.
- **TRANSCRIBED not in pystray icon flash**: `flash_processing` is only set for PROCESSING. TRANSCRIBED should not flash — verify presenter returns `flash_processing=False` for TRANSCRIBED.

## Delivery Order

1. Tray exclusivity (Phase 1) — fastest user-visible win, isolated change
2. Multi-monitor placement (Phase 2) — reliability fix, no UX model change
3. Transcript state + clipboard ownership (Phase 3) — state model + app logic
4. Blob overlay (Phase 4) — consumes TRANSCRIBED from Phase 3
